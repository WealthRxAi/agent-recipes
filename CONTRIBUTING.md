# Contributing to agent-recipes

Thanks for helping build the missing manual for AI coding agents. Every good recipe saves thousands of developers time. Here's how to add one.

## The 60-second version

1. Add your recipe object to one of the `data/part-*.json` files (or create a new one — any file matching `data/part-*.json` is picked up).
2. Run `npm run build` to regenerate `recipes.json`, the per-recipe markdown, and the site data.
3. Run `npm run validate` to check it.
4. Open a PR.

## What makes a good recipe

Recipes are **copy-paste prompts that actually work**. Before you submit, ask:

- **Specific, not generic.** "Fix TypeScript errors until clean" beats "help with TypeScript." Bad recipes describe a category; good recipes give an exact, runnable instruction.
- **Copy-paste ready.** The `prompt` should work when pasted into an agent with minimal edits. Use `{{PLACEHOLDERS}}` for the few things a user must swap in.
- **Honest and safe.** Real `safety_notes` (don't weaken types, don't commit secrets, work on a branch, no destructive deletes). No paid APIs or required API keys.
- **Concrete outcome.** `expected_output` tells the user exactly what they'll get back.

## Recipe schema

Every recipe is a JSON object with **all** of these non-empty string fields:

| Field | What it is |
|---|---|
| `id` | Unique, kebab-case, stable (e.g. `fix-ts-errors-clean`) |
| `title` | Short human title |
| `category` | One of the categories below |
| `agent` | Best-fit agent: `Claude Code`, `Codex`, `Cursor`, `Gemini CLI`, or `Any` |
| `use_case` | One sentence: who needs this and when |
| `prompt` | The copy-paste prompt (use `\n` for line breaks in JSON) |
| `safety_notes` | Real cautions — what the agent should NOT do |
| `expected_output` | What the agent should deliver |

### Allowed categories

`build-a-saas`, `fix-bugs`, `refactor-code`, `design-ui`, `write-tests`, `ship-mvp`, `create-content`, `automate-business`, `analyze-repo`, `launch-product`

### Example

```json
{
  "id": "fix-ts-errors-clean",
  "title": "Fix TypeScript errors until clean",
  "category": "fix-bugs",
  "agent": "Codex",
  "use_case": "Your build fails with TypeScript errors and you want them resolved without weakening type safety.",
  "prompt": "Goal: a clean `npx tsc --noEmit`.\n1. Run it and collect every error.\n2. Fix root causes, not symptoms...",
  "safety_notes": "Work on a branch. Never weaken tsconfig or blanket-cast to `any`.",
  "expected_output": "Zero errors from `npx tsc --noEmit` and no new `any`/`@ts-ignore`."
}
```

## Rules

- **Don't hand-edit `recipes.json`, `recipes/*.md`, or `site/recipes.js`.** They're generated by `npm run build` from `data/part-*.json`. CI rejects out-of-date generated files.
- **Keep `id` stable and unique.** People link to recipes by id.
- **No paid services or required API keys** in any recipe.
- **One concept per recipe.** Split big ideas into multiple recipes.
- **Validation must pass:** `npm run validate`.

## Project layout

```
data/part-*.json        ← source of truth (edit these)
recipes.json            ← generated index (don't edit)
recipes/*.md            ← generated per-recipe pages (don't edit)
site/                   ← static browser app (index.html + generated recipes.js)
bin/agent-recipes.js    ← zero-dependency CLI
scripts/build.js        ← regenerates recipes.json + markdown + site data
scripts/validate-recipes.js ← schema/uniqueness validation (runs in CI)
```

## Code of conduct

Be kind, be honest, assume good faith. We're all just trying to make agents less annoying to work with.
