# Write a launch-ready README

> Your project works but the README is weak; you want one that converts visitors into users and stars.

| | |
|---|---|
| **ID** | `launch-readme-polish` |
| **Category** | `launch-product` |
| **Best with** | Cursor |

## Prompt

```text
Rewrite this repo's README to be launch-ready and compelling.

1. Open with a one-line value prop and a short 'why it exists'.
2. Add a 30-second quickstart that actually works (verify commands against the repo).
3. Show real usage examples (copy-paste) and, if visual, a screenshot/GIF placeholder with a note to add it.
4. Add: features, installation, configuration table, FAQ, contributing link, license.
5. Make it skimmable with badges (build/license) and clear headings.
6. Keep it honest — no features that don't exist.

Write for a developer skimming in 10 seconds, then deciding to try it.
```

## Safety notes

Verify every command and feature against the real repo — a broken quickstart kills trust. Don't claim non-existent features or fake metrics/testimonials. Use placeholders for media that doesn't exist yet.

## Expected output

A polished README: value prop, why-it-exists, working 30-second quickstart, copy-paste examples, features, config table, FAQ, contributing/license, and badges — all accurate to the repo.
